Zend_Search_Lucene_Field

class Zend_Search_Lucene_Field

A field is a section of a Document. Each field has two parts, a name and a value. Values may be free text or they may be atomic keywords, which are not further processed. Such keywords may be used to represent dates, urls, etc. Fields are optionally stored in the index, so that they may be returned with hits on the document.

Properties

string $name Field name
boolean $value Field value
boolean $isStored Field is to be stored in the index for return with search hits.
boolean $isIndexed Field is to be indexed, so that it may be searched on.
boolean $isTokenized Field should be tokenized as text prior to indexing.
boolean $isBinary Field is stored as binary.
boolean $storeTermVector Field are stored as a term vector
float $boost Field boost factor It's not stored directly in the index, but affects on normalization factor
string $encoding Field value encoding.
